# Пара ближайших точек

## Суть задачи

Нужно найти в массиве точек такую пару, расстояние между которой минимально.

Функция должна вернуть кортеж:

```python
(distance, point1, point2)
```

## Реализация

Основной код находится в `points.py`:

- `distance(p1, p2)` — евклидово расстояние
- `closest_pair_of_points(points)` — основной алгоритм поиска

## Как работает алгоритм

Решение реализовано по схеме "разделяй и властвуй":

1. Все точки сортируются по `x`.
2. Массив делится на левую и правую половину.
3. Рекурсивно ищется лучшая пара слева и справа.
4. Затем проверяется "полоса" около линии разделения.
5. В полосе точки сортируются по `y`, и сравниваются только кандидаты, которые могут улучшить текущий минимум.

## Какие алгоритмы применены

- рекурсивное разделяй-и-властвуй
- сортировка точек
- геометрическая фильтрация по полосе около разделителя

## Сложность

В данной реализации:
- по времени: около `O(N log^2 N)`
- по памяти: `O(N)`

Дополнительный логарифмический множитель появляется из-за сортировки полосы по `y` внутри рекурсии.

## Пример использования

```python
from points import closest_pair_of_points

result = closest_pair_of_points([(1, 2), (9, 9), (9, 12)])
print(result)
```

## Что важно знать

- входной список точек сортируется на месте через `points.sort()`
- если нужно сохранить исходный порядок, лучше передавать копию списка
