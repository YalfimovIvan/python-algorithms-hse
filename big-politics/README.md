# Большая политика

## Суть задачи

Дан список численностей провинций. Нужно объединить все провинции в одну так, чтобы суммарное количество выданных новых паспортов было минимальным.

Если объединяются провинции с населением `a` и `b`, стоимость шага равна `a + b`.

## Реализация

Основная функция находится в файле `big_politics.py`:

```python
big_politics(p: list) -> int
```

Она возвращает минимальную суммарную стоимость объединения.

## Как работает алгоритм

В решении используется жадный алгоритм на минимальной куче (`heapq`):

1. Исходный список превращается в min-heap.
2. На каждом шаге извлекаются две наименьшие провинции.
3. Их сумма добавляется в общий ответ.
4. Объединённая провинция снова помещается в кучу.
5. Процесс повторяется, пока не останется один элемент.

Это тот же принцип, который используется в задаче об оптимальном слиянии файлов и в построении дерева Хаффмана.

## Почему это оптимально

Чтобы минимизировать вклад дорогих объединений в будущие шаги, выгодно как можно раньше объединять самые маленькие элементы. Именно это и делает min-heap.

## Сложность

- по времени: `O(N log N)`
- по памяти: `O(N)`

Формально в исходной учебной постановке требовалась сложность не хуже `O(N^2)`, а реализация через кучу получилась ещё эффективнее.

## Пример использования

```python
from big_politics import big_politics

print(big_politics([6, 2, 4]))  # 18
```

## Важно

Функция вызывает `heapq.heapify(p)`, то есть изменяет переданный список на месте. Если нужно сохранить исходный массив, передавайте копию:

```python
big_politics(p.copy())
```
